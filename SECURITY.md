# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in Orcasynth, please report it privately.
**Do not open a public issue for security problems.**

Use GitHub's [private vulnerability reporting](https://github.com/dragocz1995/orcasynth/security/advisories/new)
to disclose the issue. Please include:

- a description of the vulnerability and its impact,
- steps to reproduce, and
- any relevant logs or proof-of-concept.

We will acknowledge your report and work with you on a fix and coordinated disclosure.

## Scope notes

Orcasynth spawns AI coding agents that execute commands on the host. Run it only in
environments you trust, behind authentication, and review the autonomy level and
provider configuration before granting agents broad access.
